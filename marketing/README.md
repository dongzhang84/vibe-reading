# marketing/

Founder-side artifacts for early-stage outreach. Most files in this folder
are **gitignored** because they hold real user emails or personal launch
drafts; this README and `templates/` are tracked because they're useful to
other indie founders adopting the same workflow.

## Layout

```
marketing/
├── README.md                          ← this file (tracked)
├── templates/                         ← cold-reach-out templates (tracked)
│   ├── template-a-uploaded.txt        ← for users who've uploaded ≥1 book
│   └── template-c-signedup.txt        ← for users who signed up but never uploaded
├── sent-emails.json                   ← master skip-list of every recipient (LOCAL ONLY)
├── sent-state.json                    ← legacy per-batch state (LOCAL ONLY, deprecated)
├── cold-reachout-YYYY-MM-DD.txt       ← daily generated batch (LOCAL ONLY)
└── twitter-launch.txt                 ← launch post drafts (LOCAL ONLY)
```

## Workflow (3 steps, ~30 seconds for new users since last run)

```bash
# 1. Generate today's batch from new sign-ups (auto-skips anyone in sent-emails.json)
node scripts/draft-cold-emails.mjs

#    → wrote marketing/cold-reachout-YYYY-MM-DD.txt
#    → 5 new users (Tier A: 3, Tier C: 2)
#    → Skipped (already in sent-emails.json): 33

# 2. Preview every email before sending
node scripts/send-cold-emails.mjs --dry-run

# 3. Send via your Gmail (one-by-one over SMTP, 5s between sends)
node scripts/send-cold-emails.mjs
```

`send-cold-emails.mjs` writes each successful send to `sent-emails.json`,
so re-runs and future drafts auto-skip. Safe to interrupt with Ctrl+C —
state persists per-send.

## Setup (one-time)

You need Gmail SMTP credentials in `.env.local`:

```bash
GMAIL_USER=you@gmail.com
GMAIL_APP_PASSWORD=xxxxxxxxxxxxxxxx
```

The `GMAIL_APP_PASSWORD` is a 16-character app-specific password from
[Google Account → Security → App passwords](https://myaccount.google.com/apppasswords).
You need 2-Step Verification enabled to see that page.

Sending appears in your Gmail Sent folder like a normal message. No BCC,
no "via SendGrid" footer.

## Filtering options

```bash
# Only users who signed up since a date
node scripts/draft-cold-emails.mjs --since 2026-05-09

# Only Tier A (uploaded a book) or only Tier C (signed up, no upload)
node scripts/draft-cold-emails.mjs --tier A
node scripts/draft-cold-emails.mjs --tier C

# Cap the batch
node scripts/draft-cold-emails.mjs --max 20

# Send just a slice (1-indexed within the txt's numbering)
node scripts/send-cold-emails.mjs --from 8 --to 10

# Slow down pacing (default 5s between sends)
node scripts/send-cold-emails.mjs --delay-sec 10

# Use a specific batch file (defaults to today's date)
node scripts/send-cold-emails.mjs --file marketing/cold-reachout-2026-05-09.txt
```

## First-name rendering rules (kept consistent across batches)

The draft script applies these to whatever Auth metadata shows:

- **Western / pinyin name** → capitalize first letter: `alex` → `Hi Alex，`
- **Chinese name** → drop the surname (first character), keep given name:
  `林恩超` → `Hi 恩超，` / `王东` → `Hi 东，`
- **Single-character surname only** (e.g. `wang` alone, no given name) → fallback
- **Autogenerated junk** (`Gmail-All`, single letters, missing) → fallback
- **Fallback greeting** → `亲爱的朋友，`

Edit `renderGreeting()` in `scripts/draft-cold-emails.mjs` to tune.

## What about template language / tone?

Templates are intentionally written for a Chinese-speaking audience —
that's where Vibe Reading's early users came from. Translate or tone-shift
freely; the templates only need to keep the `{{GREETING}}` placeholder so
the draft script can substitute.
